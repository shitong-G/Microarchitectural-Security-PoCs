# PACMAN (ARM / Apple silicon)

## Idea

**PACMAN** (2022 IEEE S&P): defeats **pointer authentication** on **Apple M1** by combining **speculative execution** with **PAC** verification gadgets — **not** an x86 attack.

## Why there is no x86 PoC here

This repository targets **x86-64** teaching PoCs. **PAC** / **MTE** are **ARM**-centric mechanisms.

Search for **PACMAN: Attacking Arm Pointer Authentication** (2022).
